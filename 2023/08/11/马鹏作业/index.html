理论

StringBulider概念：

  • 跟string不同，StringBulider是一个可变字符序列的字符容器类。无论如何改变当前StringBulider对象的值，他都会跟着改变，返回值可以选择接受也可选择不接受，而string是不可变的，也就是说，每次创建一个string后，对其进行更改，并不会改变其原有的值。
  • 可以在StringBulider的有参构造中传入一个string类型的值，进而达到string转StringBulider的效果。也可通过调用StringBulider中append方法拼接空字符串方式进行两者转换，还可以通过StringBulider中subString方法将StringBulider转换为String，也可以直接通过string的有参构造传入StringBulider进行转换。
  • StringBuffer中定义了和StringBulider一摸一样的方法，但是StringBuffer中的方法都以Sychonized关键字进行修饰，是线程安全的，但是效率很低，StringBulider线程不安全，但是效率较高。

算法：

  • 二分查找大法：实例代码

  • 前提：利用二分法查找的数组必须是有序的。

  •  public static int select(int[] arr,int num){
     //定义三个指针
     //min表示最小索引位置
     //max表示最大索引位置
     //mid表示两者居中索引位置
        int min=0;
        int max=arr.length-1;
        int mid=(min+max)/2;
        while (min<=max){
            //如果发现当前要查找的数据比居中值小
            //此时将max最大索引移动到居中值的左边索引处
            if(arr[mid]>num){
                max=mid-1;
                mid=(min+max)/2;
                //如果查找数据大，
                //则min最小索引移动到居中值右边索引处
            } else if (arr[mid]<num) {
                min=mid+1;
                mid=(min+max)/2;
            }else {
                //两者都不满足证明等于
                //返回当前居中值
                return mid;
            }
        }
     //在min最小索引大于max最大索引的情况下，证明没找到返回-1
        return -1;
    }
